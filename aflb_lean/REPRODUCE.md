# REPRODUCE

Step-by-step instructions to independently reproduce the v51 build result.

## 1. Toolchain

This package is pinned to Lean 4.14.0 via `lean-toolchain`. If you have
`elan` installed, `lake`/`lean` will auto-select the right version. If using
the pre-pinned toolchain in this environment:

```sh
export PATH=/opt/lean4/lean-4.14.0-linux/bin:$PATH
lake --version
lean --version
```

## 2. Build

From the package root (the directory containing `lakefile.lean`):

```sh
lake build AflbLean
```

**Important:** a bare `lake build` (no target) silently no-ops in this
project — there is no default target configured. Always pass the explicit
target `AflbLean`.

This should complete with no errors and no `sorry` warnings.

## 3. Run the full integrity check

```sh
bash scripts/rebuild.sh
```

This regenerates `docs/build.log` and `docs/axioms.txt` and runs all
integrity checks (no-sorry, no-placeholder, headline-axiom-only). It should
print `=== FINAL RESULT: PASS ===` at the end.

## 4. Inspect axiom dependencies directly

```sh
lake env lean scripts/PrintAxioms.lean
```

Compare the output to `docs/axioms.txt`; they should be identical after
step 3.

## 5. Build the legacy (non-headline) layer independently

```sh
lake build AflbLean.Legacy.Conditional
```

This builds `Legacy/Hypotheses.lean` and `Legacy/Conditional.lean` (and
transitively whatever else is needed); it is not part of the headline
target and is not required for the headline result.

## 6. Verify file hashes (optional)

```sh
sha256sum -c SHA256SUMS
```

from the package root, after extracting the distributed archive (excludes
`.lake/` build artifacts, which are not part of the source distribution and
are regenerated by step 2).
