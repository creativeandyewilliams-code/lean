SCOPE

This certificate attests to a claim-status routing skeleton for the
pre-validation admission boundary in risk analysis (v81/v82). It formalizes
status non-entailments and the non-decomposability / assessment-regress
boundary as typed distinctions. It does NOT prove empirical risk, policy
efficacy, reviewer behavior, propagation efficacy, AI-risk claims, Great

are encoded as theorems (provesExcludedClaim c = false for every excluded
claim).


0. CHANGES FROM v81 (this revision)
------------------------------------
The v81 source as uploaded did NOT pass a Lean kernel build: `lake build`
failed with exit code 1 on two theorems. A simulated peer review (ChatGPT)
independently flagged both the build-fragility issue and a separate
semantic concern; both were verified against a real kernel build and fixed
here. RoutingSkeleton.lean in this v82 package therefore differs from the
originally uploaded pause_paper_v81_routing_skeleton_full.lean in exactly
two ways:

  (a) PROOF FIX (no statement changed). `fixed_point_requires_type_preservation`
      and `fixed_point_requires_independence` used `cases a; simp [...] at h ⊢;
      exact h`, which fails to type-check: `cases a` destructs the structure
      into fresh field variables, after which `h` (stated in terms of the
      original field projection) no longer matches the rewritten goal, so
      `exact h` is a type mismatch. Fix: drop `cases a` and the `at h ⊢` form;
      `simp [fixedPointGate, h]` rewrites the goal directly using `h` as a
      simp lemma against the unreduced field projection. THEOREM STATEMENTS
      ARE UNCHANGED. Verified independently by direct `lake build` (see
      Section "BUILD-FAILURE FINDING" below) before applying the fix.

  (b) SEMANTIC FIX. `wordingObjectionStatus` previously routed
      `wordingRoutesToRiskShape o = true` directly to `RoutedStatus.projectionDefeat`.
      This conflicts with the paper's own stated principle (object
      preservation before objection evaluation): crossing the risk-shape
      gate with a stable target should make a wording objection
      risk-shape-relevant and route it onward for review, not automatically
      defeat the projection. Fix: added a new `RoutedStatus.riskShapeReview`
      constructor; `wordingObjectionStatus` now returns `riskShapeReview` in
      the gated branch instead of `projectionDefeat`. Added one new theorem,
      `wording_risk_shape_not_automatic_projection_defeat`, stating that a
      gated wording objection's status is never `projectionDefeat`. The
      false-branch behavior (`canonical_weak_wording_status_carrier`) is
      unaffected.

  Theorem count: 41 -> 42. RoutedStatus constructors: 17 -> 18.
  pause_paper_v81_routing_skeleton_full.lean itself was left unmodified;
  only the copy inside this v82 build package was changed.
